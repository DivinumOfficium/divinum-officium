Generating .html files

You may download Hofficium.zip file. The file is 15.2 MB. Last update 11-03-2011

The file unzips into Hofficium folder, creating the exe file and horas subfolder with the databases. Officium.exe executable file generates plain html files, either one file for each day, or one folder for a day into a 'year/mm-dd/' folder system, 8 .html files in each folder, (one for each hora); also optionally .wav files for chanting the psalms and some hymns; also optionally the .gif images for the notes of the chants may be incorporated. The parameters, the options and the intervall for the generated day may be set up. Click Help button on the dialog table for details.

Also you may download Hmissa.zip file. The file is 6.1 MB. Last update 11-03-2011

The file unzips into Hofficium folder, creating the exe file and horas subfolder with the databases. Missa.exe executable file generates plain html file, one file for each day into $year folder. The parameters, the options and the intervall for the generated days may be set up.

Standalone program for Windows systems

You may also download Officium.zip file as the stand alone version of Divinum Officium, for Windows operating systems, which optionally speaks the text and is capable to chant the psalms and some hymns. If the call (officium.exe is started with 'notes' inline parameter, the program generated the notations for each psalm and some hymns. The file is 21.8 MB, Last update 11-03-2011

This version works without internet connection, as a regular exe program. If you unzip it it will establish an Officium folder, and in that officium.exe executable program. Running ordo.bat batch file, will show the ordo generated for the selected year, month and version. This is perpetual within the Gregorian calendar. Using My Computer, make a shortcut(s) and put it to your desktop.

It is important to leave Ahoras.ini file and horas folders in the same location. The databases are stored in horas folder together with some images and with the help files.

Also you may also download Missa.zip file as the stand alone version of Sancta Missa, for Windows operating systems, which optionally speaks the text The file is 22.2 MB, Last update 11-03-2011

This version works without internet connection, as a regular exe program. If you unzip it it will establish an Missa folder, and in that missa.exe executable program. Using My Computer, make a shortcut and put it to your desktop.

It is important to leave Amissa.ini file and horas folder in the same location. The databases are stored in horas folder together with some images and with the help files.

Voice and/or chant for the above programs

Downlod mbr folder (32,662 Kb, last updated 12-13-2010) and unzip it as sibling for OfficiumNNN or Hofficium folder, to let the program read aloud or chant the psalms and some hymns. This feature at present stage works only for windos operating systems.

For read aloud you must install setup-espeak.exe from mbr folder or downloaded from espeak stable or from espeak latest site. Install at least mb-la1 en-r and hu optional languages when asked. Additional Microsoft voices may be downloaded and install from SAPI5.1 SDK The installed voices will add automaticaly the named voice variation to SAPI5 voicelist. You may select eSpeak-HU for Latin too; the program makes the pronuntiation changes.

Also for the proper SAPI for windows version you may have to install setupreadplease2003.exe from mbr folder or downlod from Readplease2003 site. You need only the SAPI driver, so either the free of trial versions can be installed. At this stage chanting is possible only for Latin and Magyar. For chanting part of the office MBROLA speech synthesiser is used. The necessary files are stored into mbr subfolder, but you may also install from mbrola site (click on 'download' and on the mandatory 'MBROLA binary and voices' links);

Any time after rebooting the computer the MBROLA license appears, before singing, answer 'I agree'. Regardless weather you inslall it or not, mbr folder as sibling of Officennn or Hoffice folders is necessary for approve the voice selections.

Standalone program for non Windows systems

Officium.exe program is developed to run on Windows XP Vista operating systems.

For other operating systems the appropriate perl interpreter had to be downloaded and installed first, so the program could run from the symbol source = Lofficium.pl in officiumprog folder.

In Ahoras.ini ASCII text file the first line contains the necessary reference to the database in the same folder (the content is different from the one in the root directory. Also the third line of this file had to be corrected to point to a browser program, which will be called to show the help and informatory files

Symbol files
  • starting with capital A or L are the specials for standalone version,
  • their nameshakes w/o the capital letter are the specials for the web,
  • the rest are used by both versions.

For the web horas.ini file holds the reference to the database, the default set is that officiumprog folder subfolder of the cgi-bin folder, and horas folder as is with all the subfolders is in the www folder.
